Claude Code

소개 및 설치

Claude Code는 Anthropic이 만든 CLI 기반 AI 코딩 도구입니다.

설치

npm install -g @anthropic-ai/claude-code

API 키 설정

Anthropic 콘솔에서 API 키를 발급받은 후:

# 환경변수 설정
export ANTHROPIC_API_KEY=sk-ant-xxxxx

# 또는 실행 시 자동 로그인
claude

첫 실행

# 프로젝트 폴더에서 실행
cd my-project
claude

프로젝트 폴더에서 claude 명령어를 실행하면 대화형 세션이 시작됩니다.

Claude Code

기본 사용법

일상적으로 사용하는 핵심 기능들입니다.

주요 명령어

  • /help - 도움말 보기
  • /clear - 대화 초기화
  • /compact - 컨텍스트 요약
  • /cost - 사용량 확인

파일 작업 요청

# 파일 수정 요청
"login 함수에 에러 핸들링 추가해줘"

# 새 파일 생성
"사용자 인증 미들웨어 만들어줘"

# 버그 수정
"이 에러 원인 찾아서 고쳐줘"

권한 모드

  • 읽기 - 파일 읽기, 검색 (자동 허용)
  • 쓰기 - 파일 수정, 생성 (확인 필요)
  • 실행 - 셸 명령 실행 (확인 필요)
Claude Code

고급 기능

생산성을 높이는 고급 활용법입니다.

비대화형 모드

# 한 번에 명령 실행
claude -p "이 프로젝트의 구조 설명해줘"

# 파이프 입력
cat error.log | claude -p "이 에러 분석해줘"

CLAUDE.md 설정

프로젝트 루트에 CLAUDE.md 파일을 만들어 프로젝트 컨텍스트를 제공합니다:

# CLAUDE.md
이 프로젝트는 Next.js 14 + Supabase 기반입니다.
- 컴포넌트는 src/components에 위치
- API 라우트는 src/app/api에 위치
- Supabase 클라이언트는 src/lib/supabase.ts

MCP 서버 연동

외부 도구를 MCP(Model Context Protocol)로 연결할 수 있습니다.

# .claude/settings.json
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": ["-y", "@supabase/mcp-server"]
    }
  }
}
GitHub

소개 및 설정

GitHub는 Git 기반의 코드 호스팅 및 협업 플랫폼입니다.

Git 설치 확인

git --version

초기 설정

# 사용자 정보 설정
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 기본 브랜치명 설정
git config --global init.defaultBranch main

SSH 키 설정

# SSH 키 생성
ssh-keygen -t ed25519 -C "your@email.com"

# 공개키 복사 후 GitHub > Settings > SSH Keys에 등록
cat ~/.ssh/id_ed25519.pub

GitHub CLI (gh) 설치

# macOS
brew install gh

# Windows
winget install GitHub.cli

# 로그인
gh auth login
GitHub

저장소 관리

저장소 생성, 클론, 브랜치 및 PR 관리입니다.

저장소 생성 및 클론

# 새 저장소 생성 (GitHub CLI)
gh repo create my-project --public --clone

# 기존 저장소 클론
git clone git@github.com:username/repo.git

# 로컬 프로젝트를 GitHub에 연결
git init
git remote add origin git@github.com:username/repo.git
git push -u origin main

브랜치 관리

# 브랜치 생성 및 이동
git checkout -b feature/new-feature

# 브랜치 목록
git branch -a

# 브랜치 삭제
git branch -d feature/old-feature

Pull Request

# PR 생성 (GitHub CLI)
gh pr create --title "Add new feature" --body "설명"

# PR 목록 확인
gh pr list

# PR 머지
gh pr merge 123

자주 쓰는 명령어

# 상태 확인
git status

# 변경사항 스테이징 및 커밋
git add .
git commit -m "feat: add new feature"

# 원격 저장소에 푸시
git push origin main
GitHub

GitHub Actions

CI/CD 자동화 워크플로우를 설정합니다.

기본 구조

프로젝트 루트에 .github/workflows/ 폴더를 만들고 YAML 파일을 작성합니다.

배포 워크플로우 예시

# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run build
      - run: npm test

테스트 자동화

# .github/workflows/test.yml
name: Test

on:
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm test

Secrets 설정

Settings > Secrets and variables > Actions에서 환경변수를 추가합니다.

# 워크플로우에서 사용
env:
  SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
  SUPABASE_KEY: ${{ secrets.SUPABASE_KEY }}
Supabase

소개 및 설정

Supabase는 PostgreSQL 기반의 오픈소스 백엔드 서비스입니다.

프로젝트 생성

  1. supabase.com 접속 후 회원가입
  2. New Project 클릭
  3. 프로젝트명, 비밀번호, 리전 설정

클라이언트 설치

npm install @supabase/supabase-js

초기화

import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  'https://xxxxx.supabase.co',  // Project URL
  'eyJxxxxx'                     // Anon Key
)
Supabase

데이터베이스

PostgreSQL 기반 CRUD 및 실시간 기능입니다.

데이터 조회 (SELECT)

const { data, error } = await supabase
  .from('posts')
  .select('*')
  .order('created_at', { ascending: false })
  .limit(10)

데이터 삽입 (INSERT)

const { data, error } = await supabase
  .from('posts')
  .insert({ title: '제목', content: '내용' })
  .select()

데이터 수정 (UPDATE)

const { data, error } = await supabase
  .from('posts')
  .update({ title: '수정된 제목' })
  .eq('id', 1)
  .select()

데이터 삭제 (DELETE)

const { error } = await supabase
  .from('posts')
  .delete()
  .eq('id', 1)

실시간 구독

supabase
  .channel('posts')
  .on('postgres_changes',
    { event: '*', schema: 'public', table: 'posts' },
    (payload) => console.log(payload)
  )
  .subscribe()
Supabase

인증 (Auth)

이메일, OAuth 등 다양한 인증 방법을 지원합니다.

이메일 회원가입

const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'password123'
})

로그인

const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password123'
})

OAuth (Google 등)

const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'google'
})

로그아웃

await supabase.auth.signOut()

현재 사용자 확인

const { data: { user } } = await supabase.auth.getUser()

RLS (Row Level Security)

Supabase 대시보드 > SQL Editor에서 정책을 설정합니다:

-- 본인 데이터만 조회 가능
CREATE POLICY "Users can view own data"
ON posts FOR SELECT
USING (auth.uid() = user_id);
Vercel

소개 및 배포

Vercel은 프론트엔드 프로젝트를 위한 배포 플랫폼입니다.

Vercel CLI 설치

npm install -g vercel

배포 방법 1: CLI

# 로그인
vercel login

# 배포 (프로젝트 폴더에서)
vercel

# 프로덕션 배포
vercel --prod

배포 방법 2: Git 연동

  1. vercel.com 에서 Import Project
  2. GitHub 저장소 선택
  3. 프레임워크 자동 감지 후 Deploy 클릭
  4. 이후 git push 시 자동 배포
Vercel

설정 및 도메인

프로젝트 설정과 커스텀 도메인 연결입니다.

vercel.json 설정

{
  "buildCommand": "npm run build",
  "outputDirectory": ".next",
  "framework": "nextjs",
  "rewrites": [
    { "source": "/api/:path*", "destination": "/api/:path*" }
  ]
}

커스텀 도메인 연결

  1. Vercel 대시보드 > Settings > Domains
  2. 도메인 입력 후 Add
  3. DNS 설정에서 CNAME을 cname.vercel-dns.com으로 지정
  4. SSL 인증서 자동 발급
Vercel

환경변수 관리

환경별 변수 설정과 보안 관리입니다.

대시보드에서 설정

Settings > Environment Variables 에서 추가:

  • NEXT_PUBLIC_SUPABASE_URL - Supabase 프로젝트 URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY - Supabase Anon Key
  • SUPABASE_SERVICE_ROLE_KEY - 서버 전용 키 (공개 금지)

CLI로 설정

# 환경변수 추가
vercel env add SUPABASE_SERVICE_ROLE_KEY

# 환경변수 목록 확인
vercel env ls

# 환경변수 가져오기 (.env.local 생성)
vercel env pull

환경별 구분

  • Production - 프로덕션 배포에만 적용
  • Preview - PR 프리뷰 배포에 적용
  • Development - vercel dev 실행 시 적용
